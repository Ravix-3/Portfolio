Step-by-step guide

Building your
Portfolio

A structured approach — from planning what goes on the page to writing the CSS that makes it unforgettable.

01
Before you write a line of code
Plan what goes on the page
Most beginners open a text editor immediately. The ones who build good portfolios plan first.
+
1
Answer the 3 core questions
Who will read this? (recruiters, clients, devs) · What action should they take? (hire, contact, follow) · What do you want to be known for?
2
Collect your raw material first
Write your bio, list your 3–5 best projects, collect screenshots, and draft your contact info — before touching CSS. Empty sections kill momentum.
3
Choose one aesthetic direction and commit
Pick: minimal/white space · dark/tech · editorial/typographic · bold/colorful. Half-committed aesthetics look amateur. One direction, executed fully.
4
Sketch the layout on paper first
A 5-minute pencil sketch saves hours of CSS rewrites. Draw boxes for each section — don't think about colors yet, just structure.
02
Project setup
File structure & HTML skeleton
Set up clean, organized files from day one — bad habits in structure cause pain later.

Define CSS variables before writing any styles
/* variables.css — your design system in one file */ :root { /* Colors */ --color-bg: #0f0f13; --color-surface: #1a1a22; --color-text: #e8e6f0; --color-muted: #6b6880; --color-accent: #7cfc9f; /* Typography */ --font-display: 'Syne', sans-serif; --font-body: 'Fraunces', serif; /* Spacing scale */ --space-sm: 8px; --space-md: 16px; --space-lg: 32px; --space-xl: 64px; /* Max width */ --container: 1000px; }

Content architecture
What goes in each section
Every section has a job. Know what that job is before you style it.
+
⚡
Hero
Your name, one-line title, and a single CTA button. Nothing else. Recruiters decide in 3 seconds.
Must have Name · Role · CTA
👤
About
2–3 sentences max. Photo optional. Answer: who you are, what you build, and what drives you.
Include Bio · Photo · Personality
🛠

Don't use progress bars — they're meaningless. Use tag pills or grouped lists by category.
Include Tags · Categories
🗂

3–5 projects max. Each needs: screenshot, title, one-line desc, tech stack tags, and 2 links (live + GitHub).
Must have Screenshot · Links
✉️

A single mailto link is enough. Add GitHub, LinkedIn, and Twitter/X. Keep it dead simple.
Must have Email · GitHub
🧭
Navigation
Sticky or fixed nav. Logo/name on left, 4–5 links on right. Smooth scroll to section anchors.
Optional Sticky · Smooth scroll
04
Core CSS skills
CSS concepts you will use
These are not optional. Every section of a portfolio exercises these. Learn them in this order.
+
Box model
padding, margin, border, width. Every element on the page.
Flexbox
Nav bar, hero layout, skills tags row, project card footer.
CSS Grid
Projects card grid, skills category layout, about two-column.
Position
Sticky nav, fixed CTA button, absolute badges on cards.
Custom props
--color-accent, --font-body. Define once, use everywhere.
Transitions
Hover effects on buttons, links, cards. 200–300ms ease.
:hover / :focus
Button hover states. Always style :focus for accessibility.
Pseudo-elements
::before for decorative lines, dividers, accent marks.
z-index
Keep nav above content. Use a scale: 10, 100, 1000.
overflow: hidden
Clip images in cards. Prevent horizontal scroll on mobile.
Build your nav bar — the first real CSS challenge
/* A clean sticky nav — practice box model + flexbox */ nav { position: sticky; top: 0; z-index: 100; display: flex; justify-content: space-between; align-items: center; padding: 16px 40px; background: var(--color-bg); border-bottom: 1px solid rgba(255,255,255,0.08); } nav a { font-family: var(--font-body); font-size: 14px; color: var(--color-muted); text-decoration: none; transition: color 0.2s ease; } nav a:hover { color: var(--color-accent); }
05
Visual identity
Typography — the single biggest impact
Font choice is 60% of your portfolio's personality. Don't use Inter. Don't use Arial. Pick something with character.
+
1
Use a font pairing — one display, one body
Display font: for your name and section headings — make it big, make it yours. Body font: for descriptions, bio, and paragraph text — should be highly readable.
Syne
Geometric display. Strong personality. Great for tech portfolios.
Display
Fraunces
Optical size serif. Elegant and warm. Pairs with anything geometric.
Body
JetBrains Mono
For code snippets, tech tags, and labels. Instantly signals developer.
Mono
Clash Display
Variable display font. Very modern. Use for hero headline only.
Display
/* Type scale — define once, use everywhere */ :root { --text-xs: 11px; /* labels, tags */ --text-sm: 14px; /* body text */ --text-md: 18px; /* section intros */ --text-lg: 28px; /* section headings */ --text-xl: 56px; /* hero name */ --text-2xl: 88px; /* hero name — large screens */ } h1 { font-family: var(--font-display); font-size: clamp(48px, 10vw, 88px); /* fluid! */ line-height: 0.95; letter-spacing: -0.03em; }
06
Mobile-first
Responsive design approach
Recruiters check portfolios on phones. If it breaks below 390px, you've already lost.
+
1
Write mobile styles first, desktop second
Default styles = mobile (single column, full width). Then use min-width media queries to enhance for larger screens. This is mobile-first.
2
Use only 2 breakpoints for a portfolio
640px (tablet) and 1024px (desktop) are enough. Don't overthink it. Most portfolios only need one breakpoint.
3
Use clamp() for fluid typography
Instead of writing media queries just for font sizes, use clamp(min, fluid, max). It scales smoothly between any screen size.
/* Projects grid — mobile first */ .projects-grid { display: grid; grid-template-columns: 1fr; /* 1 col on mobile */ gap: 20px; } @media (min-width: 640px) { .projects-grid { grid-template-columns: repeat(2, 1fr); /* 2 cols */ } } @media (min-width: 1024px) { .projects-grid { grid-template-columns: repeat(3, 1fr); /* 3 cols */ } } /* Or use auto-fit (no media queries needed!) */ .projects-grid { grid-template-columns: repeat(auto-fit, minmax(280px, 1fr)); }
07
Finishing touches
Polish — what separates good from great
Polish is the gap between "it works" and "it's impressive". These are the details that matter.
+
1
Add a page load animation
Staggered fade-in on sections as they enter the viewport using IntersectionObserver + CSS animation-delay. One animation done well beats ten mediocre ones.
2
Hover states on every interactive element
Every button, link, card, and icon needs a hover state. Use transition: all 0.2s ease. This is the #1 thing that makes a portfolio feel alive.
3
Consistent spacing rhythm
Use multiples of 8px everywhere (8, 16, 24, 32, 48, 64, 96). Inconsistent spacing is the most common beginner mistake — it looks "off" even if you can't explain why.
4
Add a custom scrollbar
A one-liner that shows you care. ::-webkit-scrollbar with your accent color makes the page feel fully designed.
/* Scroll-triggered fade-in — JS + CSS */ .section { opacity: 0; transform: translateY(24px); transition: opacity 0.6s ease, transform 0.6s ease; } .section.visible { opacity: 1; transform: translateY(0); } /* In JS */ const observer = new IntersectionObserver(entries => { entries.forEach(e => { if (e.isIntersecting) e.target.classList.add('visible'); }); }, { threshold: 0.1 }); document.querySelectorAll('.section').forEach(s => observer.observe(s)); /* Custom scrollbar */ ::-webkit-scrollbar { width: 6px; } ::-webkit-scrollbar-track { background: var(--color-bg); } ::-webkit-scrollbar-thumb { background: var(--color-accent); border-radius: 3px; }

Pre-launch
Launch checklist
Don't publish until every item is checked. These are the things that actually get you hired.

GitHub link opens your real profile

favicon.ico added (even a simple one)

No placeholder text left ("Lorem ipsum")

Deployed to Vercel, Netlify, or GitHub Pages
